Banco de Dados


É uma coleção organizada de dados, estruturados ou não, armazenados eletronicamente para uso posterior. Esses dados podem ser de qualquer tipo, desde informações pessoais até registros de vendas, dados científicos, etc. Ele serve como um repositório centralizado para dados, permitindo que sejam acessados e manipulados por diversos usuários e aplicações.





Tipos de banco de Dados


Relacional (SQL):


Organiza e armazena dados em tabelas (entidades). Essas tabelas são compostas por linhas (tuplas), colunas (atributos), chaves primárias e estrangeiras. A característica principal dos deles é a capacidade de estabelecer relações entre essas tabelas, criando um modelo de dados mais completo e flexível.


Exemplos: Oracle, MySQL e PostGreSQL.



Não Relacional (NoSQL):


São sistemas de gerenciamento de dados projetados para lidar com grandes volumes de dados, alta escalabilidade e flexibilidade, especialmente quando os dados não se encaixam perfeitamente em um modelo tabular rígido, como os bancos de dados relacionais.


Exemplos: MongoDB, Cassandra e Redis





Diagrama conceitual - DER


É uma representação abstrata de um sistema do mundo real. Ele usa entidades, atributos, relacionamentos e a cardinalidade. É usado na fase inicial para definir necessidades, sem especificações.


Cardinalidade: exemplo: 1, n;

Entidades são retratadas por retângulos;

Relacionamento são retratadas por losangos;

Atributos são características das entidades, por exemplo, nome.





Diagrama lógico - MER


É uma representação mais detalhada de um sistema do que o modelo conceitual, mais especificado. Ele serve como uma ponte entre o mundo real (representado pelo modelo conceitual) e o mundo digital (representado pelo modelo físico).


Mesma características do DER, o que muda é os atributos, que são mais detalhados.




Dicionário de Dados


É um repositório centralizado que armazena metadados detalhados sobre os elementos de um banco de dados. Ele serve como um catálogo completo, fornecendo informações precisas sobre cada campo, tabela, relacionamento, além de regras de negócio e outras informações relevantes.
