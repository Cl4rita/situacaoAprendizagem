DHCP


É responsável por atribuir automaticamente endereços IP e outras informações de configuração de rede a dispositivos que se conectam a uma rede.


Ele simplifica a configuração de dispositivos em uma rede, eliminando a necessidade de configurar manualmente cada dispositivo.


Quando um dispositivo se conecta à rede, ele envia uma solicitação DHCP ao servidor DHCP, que responde com um endereço IP e outras configurações necessárias.

DNS


Domain Name System (DNS) é um serviço de diretório hierárquico distribuído que traduz nomes de domínio legíveis por humanos em endereços IP numéricos que identificam dispositivos na internet. Essencialmente, ele funciona como uma agenda telefônica global para a internet, permitindo que os usuários digitem nomes de domínio amigáveis (como URL) em vez de longas sequências de números (como 172.217.11.14).


Analogia:

Imagine o DNS como uma lista telefônica mundial. Quando alguém quer ligar para alguém, terá uma lista telefônica procurando o nome da pessoa que encontra o número de telefone dela. O DNS faz o mesmo, mas com endereços de sites na internet.






HTTP


O HTTP (Hypertext Transfer Protocol) é o protocolo fundamental que permite a comunicação entre navegadores e servidores web, possibilitando a exibição de páginas da internet. Quando você digita um endereço em seu navegador, ele envia uma solicitação HTTP para o servidor, que por sua vez envia o código da página de volta. O HTTP é como a linguagem que os computadores usam para "conversar" e mostrar o conteúdo da web.


Ele é a base da World Wide Web, permitindo que você navegue por sites, baixe arquivos e interaja com diversas plataformas online. Para garantir a segurança das informações transmitidas, existe o HTTPS, uma versão do HTTP que utiliza criptografia para proteger seus dados.






Conceito de Servidor


Um servidor é um dispositivo de hardware ou software que fornece serviços a outros dispositivos em uma rede. Ele opera como um host, atendendo a solicitações de clientes e fornecendo recursos como arquivos, dados, websites, e-mail e outros serviços de rede. Em termos técnicos, um servidor executa um sistema operacional especializado e software de servidor que permite a ele ouvir e responder a solicitações de rede.


A arquitetura cliente-servidor é fundamental para a maioria das aplicações de rede, desde pequenas redes locais até a internet, e envolve a divisão de responsabilidades entre o servidor (que fornece os serviços) e os clientes (que consomem esses serviços).






Conceito de TCP/IP


TCP (Transmission Control Protocol): É responsável por garantir que os dados sejam enviados de forma confiável e ordenada entre dois dispositivos. Imagine enviar uma carta: o TCP se certifica de que a carta chega completa e na ordem correta ao destinatário.


IP (Internet Protocol): É responsável por endereçar cada dispositivo na rede, como se fosse um endereço de rua. O IP atribui um número único a cada dispositivo, permitindo que os pacotes de dados sejam enviados para o destino correto.
