DHCP


É responsável por atribuir automaticamente endereços IP e outras informações de configuração de rede a dispositivos que se conectam a uma rede.


Quando um dispositivo se conecta à rede, ele envia uma solicitação DHCP ao servidor DHCP, que responde com um endereço IP e outras configurações necessárias.

DNS


É um serviço de diretório hierárquico distribuído que traduz nomes de domínio legíveis por humanos em endereços IP numéricos que identificam dispositivos na internet. Essencialmente, ele funciona como uma agenda telefônica global para a internet, permitindo que os usuários digitem nomes de domínio amigáveis (como URL) em vez de longas sequências de números (como 172.217.11.14).





HTTP


É o protocolo fundamental que permite a comunicação entre navegadores e servidores web, possibilitando a exibição de páginas da internet. Quando você digita um endereço em seu navegador, ele envia uma solicitação HTTP para o servidor, que por sua vez envia o código da página de volta. Para garantir a segurança das informações transmitidas, existe o HTTPS, uma versão do HTTP que utiliza criptografia para proteger seus dados.





Conceito de Servidor


Um servidor é um dispositivo de hardware ou software que fornece serviços a outros dispositivos em uma rede. Ele opera como um host, atendendo a solicitações de clientes e fornecendo recursos como arquivos, dados, websites, e-mail e outros serviços de rede. Em termos técnicos, um servidor executa um sistema operacional especializado e software de servidor que permite a ele ouvir e responder a solicitações de rede.





Conceito de TCP/IP


TCP: É responsável por garantir que os dados sejam enviados de forma confiável e ordenada entre dois dispositivos. O TCP se certifica de que a mensagem chega completa e na ordem correta ao destinatário.


IP (Internet Protocol): É responsável por endereçar cada dispositivo na rede, como se fosse um endereço de rua. O IP atribui um número único a cada dispositivo, permitindo que os pacotes de dados sejam enviados para o destino correto.

Modelo OSI


É um modelo conceitual que divide as funções de comunicação em redes em sete camadas. Ele serve como um padrão para que diferentes sistemas de computadores possam se comunicar de forma eficiente e interoperável.


Camadas:


  • 1.Física: Cabo Ethernet e Placa de Rede;

  • 2.Enlace de Dados: Ethernet e ARP;

  • 3.Rede: IP, roteadores;

  • 4.Transporte: TCP, UDP;

  • 5.Sessão: RPC, NetBIOS;

  • 6.Apresentação: JPEG, PNG;

  • 7.Aplicação: Navegadores Web e FTP.

FTP


É um computador dedicado a transferir arquivos entre diferentes dispositivos conectados à rede. Ele funciona como um repositório centralizado onde você pode armazenar, baixar e enviar arquivos.


Funcionalidade:

  • Backup de dados;

  • Transferência de arquivos;

  • Compartilhamento de arquivos.


Ele é simples de implementar e usar, é eficiente e também tem ampla compatibilidade. Infelizmente, não é tão seguro, pois não é criptografado.


  • FTP:

    Ideal para tarefas simples de transferência de arquivos em ambientes locais ou quando você precisa de controle total sobre o servidor.

  • Serviços em nuvem:

    Ideal para armazenamento, compartilhamento e colaboração em arquivos, especialmente se você precisa de acesso remoto e recursos avançados.
